CREATE TABLE IF NOT EXISTS users (
    telegram_id INTEGER PRIMARY KEY,
    username TEXT,
    client_rating REAL DEFAULT 5.0,
    executor_rating REAL DEFAULT 5.0,
    client_reviews_count INTEGER DEFAULT 0,
    executor_reviews_count INTEGER DEFAULT 0,
    balance_usdt REAL DEFAULT 0.0, -- MANDATORY ZERO INITIALIZATION
    frozen_usdt REAL DEFAULT 0.0,
    is_blocked INTEGER DEFAULT 0,
    is_shadow_banned INTEGER DEFAULT 0,
    disputes_initiated INTEGER DEFAULT 0,
    disputes_lost INTEGER DEFAULT 0,
    total_tasks_client INTEGER DEFAULT 0,
    total_tasks_executor INTEGER DEFAULT 0,
    notification_categories TEXT DEFAULT '[]', -- JSON string array of interest tags
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP

CREATE TABLE IF NOT EXISTS tasks (
    task_id INTEGER PRIMARY KEY AUTOINCREMENT,
    channel_message_id INTEGER DEFAULT NULL,
    client_id INTEGER,
    executor_id INTEGER DEFAULT NULL,
    title TEXT NOT NULL,
    description TEXT NOT NULL,
    deadline TEXT,
    category TEXT DEFAULT 'Generale',
    attachments TEXT DEFAULT NULL, -- JSON string array of string file_ids
    reward_gross REAL NOT NULL,
    reward_net REAL NOT NULL,
    status TEXT CHECK(status IN ('open', 'in_progress', 'completed', 'dispute', 'cancelled')),
    is_direct INTEGER DEFAULT 0,
    target_executor_identity TEXT DEFAULT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    claimed_at TIMESTAMP DEFAULT NULL,
    completed_at TIMESTAMP DEFAULT NULL,
    disputed_at TIMESTAMP DEFAULT NULL
);

CREATE TABLE IF NOT EXISTS deal_sessions (
    session_id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id INTEGER UNIQUE,
    client_id INTEGER,
    executor_id INTEGER,
    room_token TEXT UNIQUE,
    status TEXT CHECK(status IN ('active', 'closed', 'disputed')),
    initialized_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS deal_messages (
    message_id INTEGER PRIMARY KEY AUTOINCREMENT,
    task_id INTEGER,
    sender_id INTEGER,
    message_type TEXT,
    file_id TEXT DEFAULT NULL,
    content_preview TEXT,
    timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS admin_revenue (
    id INTEGER PRIMARY KEY DEFAULT 1,
    total_collected_fees REAL DEFAULT 0.0
);

### 5. ENHANCED DISPUTE ARBITRATION SYSTEM (handlers/admin.py)
- When either party clicks `[🚨 Apri Controversia]`:
  1. Instantly tear down the dynamic routing keys in `deal_sessions` (set status to `'disputed'`).
  2. Query the entire transaction history from `deal_messages` and compile all task data and attachments.
  3. Route the comprehensive investigative audit package straight to `ADMIN_TG_ID` along with these inline resolution hooks:
     - `[🟢 Sblocca per Esecutore]`: Triggers the atomic 90/10 escrow split. Credits 90% to the worker's balance, routes the 10% platform fee to `admin_revenue`, increments the client's `disputes_lost` metric, and closes the room.
     - `[🔴 Rimborsa Committente]`: Triggers a full transaction rollback. Restores 100% of the gross asset pool back to the client's active `balance_usdt`, sets the task to `cancelled`, increments the executor's `disputes_lost` counter, and closes the room.
- If a user's dispute-to-task ratio exceeds 30% or their `disputes_lost` count passes a threshold, auto-flag their account for manual administrative review and shadow-ban them.

### 6. BI-LATERAL PRODUCTION PAYMENTS ENGINE (handlers/payments.py)

#### Top-up Track A: CryptoBot (USDT Gateway)
- When a user inputs an amount to buy USDT via their card/bank account (Visa, Mastercard, Revolut, PostePay, etc.), make an asynchronous POST call to `https://pay.crypt.bot/api/createInvoice`.
- Pass headers: `Crypto-Pay-API-Token: {CRYPTOBOT_TOKEN}`. Pass JSON body: `asset: "USDT"`, `amount: "{amount}"`, and embed the user's `telegram_id` into the `payload` string.
- Return the `pay_url` as an inline button titled: "💳 Paga via CryptoBot P2P".

#### Top-up Track B: Telegram Stars (In-App Purchases XTR Gateway)
- When a user opts to top up via Telegram Stars, invoke the bot's native `send_invoice` method using the currency parameter `"XTR"`.
- Set the prices array based on the target USDT amount divided by the configured `STAR_TO_USDT_RATE`.
- Upon catching the successful `PreCheckoutQueryHandler` and eventual `SuccessfulPayment` notification object, isolate the token total. Calculate the asset value using `Stars * STAR_TO_USDT_RATE`. 
- Deduct the platform's 10% structural monetization fee immediately, route it to `admin_revenue`, and credit the remaining 90% directly to the user's active database row `balance_usdt`. Notify them instantly in Italian.

#### Production On-Demand Withdrawals (/prelievo)
- Implement a user-facing asset extraction command matching the layout format: `/prelievo <amount>`.
- Perform a strict validation check against the user's live database row `balance_usdt`. If the requested amount is greater than their unfrozen funds, abort and reply: "❌ Saldo insufficiente."
- If valid, immediately deduct the amount from their internal database row `balance_usdt`.
- Trigger an asynchronous POST API call to CryptoBot's secure outbound transfer endpoint: `https://pay.crypt.bot/api/transfer`. Pass parameters: `user_id: {user_telegram_id}`, `asset: "USDT"`, `amount: {validated_amount}`, and pass a unique string UUID as `spend_id` to enforce absolute request idempotency.
- If the transfer confirms successfully, finalize the database rows. If it fails, trigger an absolute transaction rollback, restore the funds back to the user's `balance_usdt`, and send a notification: "⚠️ Errore di rete CryptoBot. L'importo è stato riaccreditato al tuo saldo interno."
